# ML-модуль: подробное объяснение с нуля

## 1. Бизнес-смысл ML в проекте

ML здесь нужен не ради “магии”, а ради помощи пользователю ответить на вопрос:

**Как может измениться цена акции, если поменяются макроэкономические условия?**

Это делает систему полезной именно как учебную платформу:

- пользователь не просто смотрит график;
- он может моделировать рынок;
- видит, как факторы влияют на цену;
- учится принимать более осмысленные решения.

## 2. Тип задачи

Тип задачи:

- **многофакторная регрессия**.

Что предсказываем:

- цену закрытия актива на следующий торговый день.

## 5. Этапы ML pipeline

### Этап 1. Сбор истории

Для выбранного тикера backend:

1. Загружает 2-3 года истории по акции.
2. Загружает исторические ряды факторов.
3. Приводит даты к общему индексу.

### Этап 2. Очистка и синхронизация

Проблема:

- разные источники публикуют данные с разной частотой;
- где-то есть пропуски;
- торговые дни и календарные дни не совпадают.

Что делается:

- сортировка по дате;
- объединение по дате;
- `forward fill` и `back fill` для факторов;
- удаление пустых строк.

### Этап 3. Формирование target

Модель должна предсказывать цену вперед, а не повторять текущую.

Поэтому:

- `target = close.shift(-1)`

### Этап 4. Разделение train/test

Данные делятся по времени:

- ранняя часть — train;
- последняя часть — test.

Это важно:

- нельзя shuffle временной ряд как обычный табличный датасет;
- иначе получится data leakage.

### Этап 5. Обучение

Сейчас используется:

- `LinearRegression`

### Этап 6. Оценка

Метрики:

- `R^2`
- `MAE`
- `RMSE`

### Этап 7. Сохранение

Сохраняются:

- joblib artifact;
- coefficients;
- intercept;
- baseline means;
- train/test rows;
- training window;
- dataset CSV.

## 6. Почему модель линейная

Потому что на старте для учебной системы важны:

- интерпретируемость;
- стабильность;
- простота объяснения;
- простота внедрения;
- отсутствие лишней инфраструктуры.

Линейная регрессия хорошо подходит как baseline.

## 7. Driver analysis

Пользователю важен не только ответ “сколько будет цена”, но и:

- почему именно такой результат;
- какой фактор сильнее всего повлиял;
- знак влияния.

### Как это считается

Для линейной модели вклад признака вычисляется как:

```text
coef_i * (x_i - mean_i)
```

Где:

- `coef_i` — коэффициент модели;
- `x_i` — текущее или пользовательское значение фактора;
- `mean_i` — среднее значение фактора на train.

Это дает интерпретируемый вклад фактора в итоговую цену.

## 8. Сценарный анализ

### Что делает пользователь

Он меняет значения:

- Brent;
- USD/RUB;
- IMOEX;
- Key Rate;
- RGBI.

### Что делает backend

1. Берет текущую модель для тикера.
2. Формирует вектор признаков.
3. Подставляет новые пользовательские значения.
4. Считает новую цену.
5. Возвращает:
   - predicted price;
   - impact percent;
   - drivers;
   - confidence score.

## 9. Почему это подходит под постановку задачи

Задача требовала:

- многофакторную регрессию;
- работу с реальными рыночными данными;
- расчет сценарной цены;
- факторное объяснение результата.

Все эти части теперь есть.

## 10. Что можно улучшать дальше

### CatBoost

Можно заменить baseline-модель на:

- `CatBoostRegressor`

Это даст:

- лучшую работу с нелинейностями;
- более сильную модель на сложных зависимостях.

### SHAP

Сейчас объяснение основано на линейных вкладах. Следующий шаг:

- подключить полноценный `shap`
- строить SHAP values для каждой модели

### Дополнительные признаки

Можно добавить:

- volume;
- realized volatility;
- returns IMOEX;
- spread между bond index и key rate;
- rolling features;
- lag features.

### Более строгая валидация

Можно внедрить:

- rolling window validation;
- walk-forward validation;
- контроль economic consistency.

## 12. Что команда должна помнить

- ML здесь не изолирован от backend, а встроен в продуктовый процесс;
- контракт API важнее внутренней реализации модели;
- модель можно менять, но нельзя бездумно ломать формат ответа;
- для пользователей объяснимость важна почти так же, как и точность.
