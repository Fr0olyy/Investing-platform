# Backend: подробное объяснение с нуля

## 1. Что такое backend в этом проекте

Backend — это серверная часть системы, которая:

- хранит пользователей;
- управляет виртуальными портфелями;
- получает рыночные данные;
- проводит сделки;
- рассчитывает результаты;
- отдает данные frontend-у по HTTP API.

Frontend — это только интерфейс. Вся бизнес-логика находится здесь.

## 2. Почему FastAPI

FastAPI выбран потому, что он дает:

- быстрый REST API;
- автогенерацию Swagger;
- хорошую типизацию;
- удобную интеграцию с Pydantic;
- понятную структуру для учебного и production-подобного проекта.

## 3. Основные обязанности backend-а

### Пользовательский контур

- регистрация;
- логин;
- получение профиля.

### Инвестиционный контур

- список инструментов;
- текущие котировки;
- свечи;
- покупка/продажа;
- портфель;
- история операций.

### Инфраструктурный контур

- healthcheck;
- синхронизация рынка;
- обучение модели;
- обновление прогнозов.

## 4. База данных

### `users`

Хранит:

- email;
- пароль в хешированном виде;
- роль;
- активность.

### `portfolios`

Один пользователь = один виртуальный счет.

Хранит:

- cash balance;
- базовую валюту.

### `positions`

Текущие открытые позиции пользователя.

### `trades`

История покупок и продаж.

### `assets`

Справочник инструментов:

- тикер;
- имя;
- сектор;
- биржа;
- board;
- lot size.

### `quotes`

Снимки текущего рынка по активам.

### `candles`

Исторические дневные свечи для графиков.

### `macro_indicator_snapshots`

История макроиндикаторов:

- Brent;
- USD/RUB;
- IMOEX;
- RGBI;
- Key Rate.

### ML-таблицы

- `ml_model_metadata`
- `predictions`
- `scenario_simulations`

## 5. Поток данных в backend

### При загрузке списка активов

1. Frontend вызывает `GET /api/v1/assets`.
2. Backend берет `Asset`.
3. Для каждого `Asset` находит последнюю `Quote`.
4. Если есть — добавляет последний `Prediction`.
5. Возвращает DTO.

### При запросе графика

1. Frontend вызывает `GET /api/v1/assets/{ticker}/candles`.
2. Backend читает свечи из `candles`.
3. Возвращает историю за нужное число дней.

### При покупке

1. Приходит `ticker` и `quantity`.
2. Backend берет текущую цену.
3. Считает стоимость сделки.
4. Проверяет баланс.
5. Обновляет `Portfolio`.
6. Обновляет `Position`.
7. Создает запись в `Trade`.

## 6. Почему логика разделена на сервисы

Потому что без этого в роутерах быстро появляется:

- дублирование;
- нечитаемый код;
- сложность тестирования;
- высокий риск ошибок.

Сервисный слой делает проект устойчивым.

## 7. Реальные данные рынка

Backend уже получает настоящие данные:

- котировки акций — MOEX ISS;
- история свечей — MOEX ISS;
- индексы — MOEX ISS;
- курс доллара — ЦБ;
- ключевая ставка — ЦБ;
- Brent — FRED.

Важно:

- новости пока demo;
- рынок и ML уже real-data based.

## 8. Fallback-режим

Если внешний API недоступен:

- backend не должен перестать запускаться;
- для части функций используется seed/fallback;
- это полезно для локальной разработки и демо-сценариев.

## 9. Docker

В Docker-конфигурации:

- отдельный контейнер БД;
- отдельный контейнер backend;
- переменные окружения через `.env`;
- быстрый запуск всей команды одной командой.
