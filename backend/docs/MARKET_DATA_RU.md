# Реальные рыночные API и как они используются

## 1. Зачем нужен отдельный документ

Этот проект работает не на статичных mock-данных, а на реальных открытых источниках.

Поэтому команде важно понимать:

- откуда именно берутся данные;
- какой источник за что отвечает;
- как данные превращаются в backend-структуры;
- как эти же данные используются в ML.

## 2. MOEX ISS

### Что это

MOEX ISS — это публичный интерфейс Московской биржи для получения рыночных данных.

### Где используется

- текущая цена акций;
- история акций;
- текущие индексы;
- история индексов.

### Endpoint-ы

#### Текущая котировка акции

```text
GET /iss/engines/stock/markets/shares/boards/TQBR/securities/{ticker}.json
```

#### История акции

```text
GET /iss/history/engines/stock/markets/shares/boards/TQBR/securities/{ticker}.json
```

#### Текущий индекс

```text
GET /iss/engines/stock/markets/index/securities/{secid}.json
```

#### История индекса

```text
GET /iss/history/engines/stock/markets/index/securities/{secid}.json
```

### Какие поля берем

Для акций:

- `LAST`
- `OPEN`
- `HIGH`
- `LOW`
- `LCLOSEPRICE`
- `LOTSIZE`
- `SHORTNAME`
- `VOLTODAY`

Для истории:

- `TRADEDATE`
- `OPEN`
- `HIGH`
- `LOW`
- `CLOSE`
- `VOLUME`

Для индексов:

- `CURRENTVALUE`
- `LASTVALUE`
- `OPENVALUE`

## 4. FRED и Brent

Для Brent используется открытый CSV-ряд:

```text
https://fred.stlouisfed.org/graph/fredgraph.csv?id=DCOILBRENTEU
```

Почему так:

- простой доступ;
- подходит для автоматической загрузки;
- не требует ключа;
- удобен для pandas.

## 5. Как данные попадают в backend

### Текущий рынок

При вызове `MarketService.refresh_market_snapshot()`:

1. Для каждого актива берется текущий snapshot.
2. В БД создается запись в `quotes`.
3. Подтягивается история свечей.
4. В БД обновляется таблица `candles`.
5. Загружаются текущие макрофакторы.
6. Они сохраняются в `macro_indicator_snapshots`.

## 6. Как данные попадают в ML

При вызове `MLService.train_models()`:

1. Берется история актива.
2. Берутся все макроряды.
3. Все синхронизируется по дате.
4. Формируется табличный датасет.
5. Датасет сохраняется в `ml/datasets`.
6. Модель обучается.

## 7. Ограничения и риски

### MOEX

- возможны временные задержки;
- структура ответа зависит от конкретного market/board;
- часть полей может быть `null`.

### ЦБ

- XML и HTML форматы менее удобны, чем чистый JSON;
- ключевая ставка публикуется не как REST JSON API.

### FRED

- это открытый макроисточник, а не торговый терминал;
- возможны выходные и пропуски, которые нужно аккуратно обрабатывать.

## 8. Что можно улучшать дальше
